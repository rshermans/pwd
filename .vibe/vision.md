# Visao do Produto

## Problema

Projetos de conteudo costumam falhar por falta de:
- visibilidade de progresso;
- consistencia operacional;
- dados confiaveis para decisao.

## Solucao

Construir um "Operating System de Conteudo" com duas interfaces:
- **Homepage (`docs/`)**: orientacao, backlog, status e links de navegacao.
- **Portal (`portal/`)**: demonstracao viva dos conteudos e desempenho.

## Principios

1. **Publico por padrao**: resultados visiveis para gerar credibilidade.
2. **Baixa friccao**: atualizar rapido, sem bloqueio por infraestrutura.
3. **Dados acionaveis**: KPI serve para decisao, nao para decoracao.
4. **Evolucao incremental**: comecar sem DB, escalar quando houver sinal.

## Objetivos de 90 dias

- Entregar homepage e portal com fluxo estavel de publicacao.
- Implantar rotina semanal de medicao e decisao.
- Consolidar um estudo de caso com impacto comprovavel.
- Criar trilha de migracao para backend com PostgreSQL.
